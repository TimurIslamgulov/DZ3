Инструкция по командам git:

init - создание репозитория в папке

add - добавление файлов к отслеживанию

commit "text"- сохранение сделанных изменений

commit -am "text" - добавление файлов к отслеживанию и сохранение сделанных изменений

branch -M main - из master в main

diff - отображение изменений сделанных с предыдущего commit`а

log - вывод всех commit'ов

log --graph - список commit'ов в виде дерева (графа)

checkout <название_ветки> - переход между разными commit'ами и ветками

checkout -b <название_ветки> - создание новой ветки с одновременным переходом в нее

branch - список веток

branch name - создание ветки name

branch -d name - удаление ветки name

status - информация о текущем состоянии git

reflog - история изменений со всеми перемещениями

merge - слияние веток

remote add <название короткого пути> <ссылка на репозиторий> - создание короткой ссылки на репозиторий 

remote rm <название короткого пути который удалить> - удаление 

push -u <название короткого пути> main - перемещение в репозиторий ветки main

clone - клонирование репозитария с GITHUB.com (git clone <ссылка на репозиторий> <название папки>)

fetch - связывается с удалённым репозиторием и забирает из него все изменения, которых у вас пока нет и сохраняет их локально

pull - работает как комбинация команд git fetch и git merge, т. е. Git вначале забирает изменения из указанного удалённого репозитория, а затем пытается слить их с текущей веткой.

push - используется для установления связи с удалённым репозиторием, вычисления локальных изменений отсутствующих в нём, и собственно их передачи в вышеупомянутый репозиторий. Этой команде нужно право на запись в репозиторий, поэтому она использует аутентификацию.
