# **Работа с Git**
# 1. Подготовительные работы

Для установки Git сначала скачаем последнюю версию ПО с официального сайта  [![Скачать последнюю версию Гита](Git.png)](https://git-scm.com/downloads)

Следом скачаем и установим VSCode
[![Скачать последнюю версию VSC](VSCode.png)](https://git-scm.com/downloads)

*Прежде чем начать работу, необхоодимо проверить корректно ли установился Git, выполнив команду*

1) **git --version**

*Для удобства и безошибочной работы необходимо представиться системе с помощью комманд:*

2) __git config --global user.name "Name"__

3) __git config --global user.email name@example.com__ 

*Выполнить пару настроек для включения цветной подсветки во время ввода комманд и установки редактора по умолчанию соответственно:*

4) __git config --global color.ui auto__

5) __git config --global core.editor "[Program]"__

*Для просмотра всех наших веток и ссылок на расположения репозитория, к которому подключены, воспользуемся*

6) **git config --list**

# 2. Начало работы

_Для продолжения работы необхоимо создать репозиторий: откроем в ПО Vscode, предварительно созданную папку и выполним команду:_

* __git init__

_Создадим файл с расширением Markdown, используя произвольное название файла:_

+ __git touch Git_instructiom.md__

_Для удаления ненужного нам файла воспользуемся командой с произвольным названием файла:_

+ __git rm Git_instructiom.md__

_Cохраним нашу информацию, добавив наш файл в ~~видимые~~ отслеживаемые:_

+ __git add Git_instructiom.md__

_Для сохранения нашей информации и всех созданных файлов, необходимо воспользоваться:_

+ __git add .__

_Выполним фиксацию внесенных изменений с помощью:_

* __git commit -m "Описание изменений"__

_Выполним фиксацию внесенных изменений без испольования команды git add с помощью:_

> * __git commit -аm "Описание изменений"__

_Для просмотра последних изменений по сравнению с последней сохраненной версией документа воспользуемся:_:

* __git diff__

_Для сравнения двух коммитов, воспользуемся командой с использованием произвольных хэшей_:

* __git diff 68ce adcf__

# 3. Основные работы

_Для просмотра всех коммитов последних сохранений, воспользуемся командой:_

* __git log__

_Чтобы посмотреть какие последние изменения были сделаны, воспользуемся командой:_

> * __git log -p__

_Чтобы посмотреть краткое описание наших коммитов, воспользуемся командой:_

> * __git log --online__

_Чтобы посмотреть список коммитов в графическом виде, воспользуемся:_

> * __git log --graph__

_Если нм нужно посмотреть подробный список всех изменений, воспользуемся:_

> * __git reflog__

_Для перехода по веткам используем команду с произвольным названием ветки:_

* __git chekout main__

_В случае, если нам понадобится вернуться к предыдущим изменениям, скопируем 4 первых символа хэша выбранного коммита и добавим в конце команды:_

> * __git chekout 91ad__

! С помощью данной команды можно перемещаться так же по веткам и файлам !

_Чтобы создать ветку и перейти на ней:_

> * __git chekout -b новая ветка__

_Рекомендуется регулярно проверять статус последних изменений с помощью:_

* __git status__

_В случае имеющихся активных изменений, необходимо зафиксировать текущие изменения в виде коммита!_

_Чтобы создать новую вектку для сохранения наших коммитов, выполним:_

* __git switch -c название новой ветки__

_Для удаления ненужной нам ветки, используем:_

* __git switch -d название ветки__

_Чтобы посмотреть список всех наших веток, а так же узнать на какой мы ветке, используем:_

* __git branch__

_Самая простая команда для создания новой ветки_

>* __git branch название ветки__

_Для удаления ветки, воспользуемся:_

>* __git branch -d название ветки__

_Чтобы назначить ветку главной, используем:_

>* __git branch -m название ветки__

_Для слияния дух веток нам поможет команда:_

* __git merge название ветки, которую копируем__

! Чтобы сляние прошло успешно, нужно находиться в той ветке, в которую мы хотим внести изменения !

_Чтобы откатить слияние веток, используем:_

>* __git merge --about__

_Чтобы сбросить состояние файла на укзанное, воспользуемся:_

* __git restore__

_Для отмены изменений в рабочем репозитории, воспользуемся:_

>* __git restore имя папки__

_Чтобы перейти к состоянию нужного коммита и затереть предыдущий, воспользуемся например:_

* __git reset 47e7588__

_Для восстановления удаленного комита, перейдем в git reflog, скопируем хэш удаленного комита и, воспользуемся например:_

>* __git reset 503acf0__

_Для создания нового коммита, который отменяет действие предыдущего воспользуемся:_

* __git revert__

# 4.  Работа с удаленными серверами

_Для просмотра краткого списка подключенных соединений, воспользуемся:_

* __git remote__

_Для просмотра списка всех подключенных соединений, воспользуемся:_

>* __git remote -v__

_Для настройки удаленного подключения и синхронизации локальной папки с удаленной на github.com, воспользуемся например:_

>* __git remote add origin https://github.com/geekbrains-git/repo-github.git__

_Чтобы скачать удаленный репозиторий, необходимо пройти на сайт github.com, открыть папку в VSC и ввести например:_

* __git clone add origin https://github.com/ilnar-geekbrains/version_control_lection3__

_Чтобы отправить наши изменения в удаленный репозиторий, с которым настроена синхронизация используем:_

* __git push -u origin main__

! Где origin - это удаленный репозиторий, а main ветка из которой будут браться последние изменения. Иногда вместо origin требуется использовать origin2 и т.д. Ветку main принято считать основной при работе на github.com !

_Так же, чтобы отправить изменения с локального репозитория на удаленный можно воспользоваться:_

>* __git push__

_В случае, если команда выполнится с ошибкой, необходимо скопировать и использовать команду из текста:_

>* __git push --set-upstream origin     название нашей ветки__

_Чтобы скачать все изменения из удаленного репозитория можно воспользоваться:_

* __git pull__

! При этом произойдет слияние более новой информации с удаленного репозитория с более старой в локальном репозитории !

_Чтобы поучаствовать в каком то проекте, заходим в нужный нам репозиторий на странице владельца в бразуере по умолчанию, например:_

* __https://github.com/ilnar-geekbrains/group_903____

_Нажимаем на кнопку Fork для копирования чужого репозитория на свой аккаунт на github.com:_
![Fork](Ilnar.png)

_Далее уже по знакомой нам схеме скачиваем удаленный репозиторий, только уже со своего аккаунта, скопировав ссылку по кнопке Code:_

* __git git clone https%//github.com/MaksFromGeek/group_903.git__

_Далее необходимо перейти в папку с новым локальным репозиторием ч/з терминал, создать и перейти на ветку по известной нам команде, например:_

* __git chekout -b maks__

! Если перейти в нужную папку не удается, нужно вначале закрыть предыдущую папку в VSCode ч/з меню Файл > Закрыть папку !

_Создадим файл README.md, проверим чтобы лежал в папке group_903 и внесем в него информацию, после сохраним изменения:_

* __git commit -am "Сохранил изменения в файле реадме"__

_Чтобы перенести наши изменения в удаленный репозиторий воспользуемся знакомыми командами:_

>* __git push__

_И этой командой, в случае, если команда выполнится с ошибкой:_

>* __git push --set-upstream origin     название нашей ветки__

# 6.  Системные команды

_Посмотрим список всех файлов и папок, расположенных в нашей дирректории:_

> * __ls__

_Посмотрим список файлов и папок, расположенных в нашей дирректории, включая скрытые:_

>> * __ls -a__

_Эта команда позволяет нам очищать рабочую область терминала:_

* __clear__

_Для создания новой папки, воспользуемся:_

* __mkdir название папки__

_Чтобы перемещаться между папками, воспользуемся:_

* __cd название папки__

! Для успешного выполнения данной команды должна соблюдаться последовательность папок !

_Для создания нового файла, воспользуемся например:_

* __touch test.txt__

_Чтобы скопировать файл, воспользуемся например:_

* __cp test.txt test_copy.txt__

! Появится новый файл, включающий все данные из старого файла !

_Для переименования файла, воспользуемся например:_

* __mv test.txt max.txt__

_Чтобы записать текст в файл, воспользуемся например:_

* __echo "Hello" > test.txt__

_Для вывода на экран содержимого файла, воспользуемся например:_

* __сat test.txt__

_Для удаления файла, воспользуемся например:_

* __rm test.txt__

_С помощью данной команды можно так же удалить папку, для рекусивного удаления воспользуемся например:_

* __rm -r max__
