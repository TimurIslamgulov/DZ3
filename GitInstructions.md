## Основные команды
После установки необходимо «представиться» системе контроля версий. Это нужно сделать всего один раз, и git запомнит вас. Для этого нужно ввести в терминале 2 команды:

- git config --global user.name «Ваше имя английскими буквами»
- git config --global user.email ваша почта@example.com

Эта команда позволяет из папки сделать репозиторий

>git init

Добавить файл в Git

>git add .\filename

Добавить коммит(он сохраняет весь репозиторий, можно написать параметр -am, чтобы git добавил сразу все файлы)

>git commit -m "Заметка"

Посмотреть чем отличается текущий файл от того, что уже сохранено

>git diff

Посмотреть состояние Гита

>git status

Журнал изменений

>git log

Посмотреть ветви

>git branch

Перейти на какую-то ветвь

>git checkout branch_name 

Добавление новой ветви

git branch *name

"Слить" в одну две ветви, нужно находится в той, в которую сливается другая

>git merge

Удаление ненужной ветви

>git branch -d text_formatting

Посмотреть граф

>git log --graph

## Работа с удаленным репозиторием

Скачать репозиторий с Гитхаба:

>git clone *ссылка

Чтобы отправить изменения в ГитХаб, есть команда:

>git push

Чтобы синхронизироваться с версией с ГитХаба:

>git pull

* Чтобы поучаствовать в работе над чужим репозиторием, можно его “форкнуть”, то есть скопировать в свой список репозиториев на аккаунте
* Когда я хочу внести изменения в проект, то это всегда делается в отдельной ветке.
* После этого форка, нужно "клонировать" свою версию репозитория на свой компьютер
* Производим изменения в отдельной ветке
* ОТправляем изменения на свой аккаунт(git push)
* В Гитхабе появляется возможность pull request
