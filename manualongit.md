# Что такое GIT и основные команды для работы с ним

**GIT** — распределённая система контроля версий, созданная в 2005 году Линусом Торвальдсом, разработчиком ядра ОС Linux.

*более подробно про Git можно почитать на <https://en.wikipedia.org/wiki/Git> или  <https://habr.com/ru/post/472600/>*

#### Основные команды для работы с Git:

------------
1. Создание репозитория

     `git init `

2. Добавление отдельных файлов или всех файлов в область подготовленных файлов

     `git add `
	 
	 *(добавить отдельные файлы можно с помощью  указания конкретного имени файла **git add [имя файла]**, если же нужно добавить все файлы, то **git add [.]**)*

3. Внесение изменений однострочным сообщением (комментарием)

      `git commit -m`
	  
	  *(при создании коммита в репозитории можно добавить однострочное сообщение с помощью параметра **commit** с флагом **-m**. Само сообщение вводится непосредственно после флага, в кавычках **git commit -m "комментарий"**)*
	 
4. Добавление файлов с внесением изменений однострочным сообщением (комментарием)

      `git commit -am`
	  
	  *(данная команда объединяет в себе сразу 2 команды **git commit** и **git add**. Так же, как и в предыдущем варианте, при создании коммита само сообщение вводится непосредственно после флага, в кавычках **git commit -аm "комментарий"**)*
	  
	  <sub> *Примечание: команда **git commit - am** сработает, только если ранее уже был создан хоть один коммит с помощью команды **git commit -m*** </sub>
	  
5. Проверка статуса репозитория

      `git status`
	  
	  *(команда показывает на какой ветке мы сейчас находимся и состояние всех файлов.)*
	  
6. Переключение между  ветками

      `git checkout`
	  
	  *(Команда **git checkout** позволяет переключаться между последними коммитами веток. Для перехода к оспредененному коммиту или ветке требуется ввести после команды **git checkout** название ветки, или индивидуальный (уникальный номр коммита) **git checkout название ветки** или **git checkout 9g567..**)*
	  
7. Создание ветки и переключение в нее

      `git checkout -b`
	  
	  *(данная команда объединяет в себе 2 команды **git checkout** и **git branch**. Название создаваемой ветки вводится после флага **-b**,  __git checkout -b название ветки__)*
	  
8. Просмотр существующих веток и создание новой ветки

      `git branch`
	  
	  *(данная команда перечисляет существующие ветки, отметив активную знаком "звездочка" и/или цветовой индикацией. Для создания новой ветки,после команды **git branch** требуется ввести допольнительно название создаваемой ветки **git branch название ветки**)*
	  
9. Слияние веток

      `git merge`
	  
	  *(выполнять данную команду требуется из той ветки, в которую будут вноситься изменения или дополнения из другой ветки. Для этого после команды **git merge** прописывается имя ветки, которая будет сливаться с текущей **git merge название ветки**)*
	  
10. Удаление ветки

      `git branch -d`
	  
	  *( название ветки вводится поле флага **-d** , **git branch - d название ветки**)*
	  
11. Просмотр изменений в файлах

     `git diff`
	 
	 *(позволяет увидеть что именно изменилось в файлах. Если после команды **git diff** ввести уникальный номер коммита, то можно будет увидеть разницу текущего состояния проекта и коммита чей номер указан после команды. **git diff 5g987**)*
	 
12. Журнал изменений

     `git log`
	 
	 *(позволяет вывести журнал всех изменений. Если после команды **git log** ввести уникальные номера коммитов через троеточие, то можно будет увидеть журнал изменений с определенного коммита по определенный коммит. **git log 56d89...487li**)*
	 
	  `git log --graph`
	  
	  *(данная команда выводит все коммиты в виде древа)*
13. История перемещений и коммитов

      `git reflog`
	  
14. Создание копии (удаленного) репозитория

      `git clone`
	  
	  *(позволяет сделать копию удаленного репозитория. После введения команды **git clone** нужно указать ссылку на удаленный репозиторий который будет клонирован **git clone https://github.com/...)*
	  
15. Отправка данных на сервер

     `git push`
	 
16. Извлечение и загрузка содержимого из удаленного репозитория.

     `git fetch`
	 
	 *(команда __git fetch__ Извлекает и загружает содержимое из удаленного репозитория)*

     `git pull`
	 
	 *(команда __git pull__ содержит в себе функции 2 команд **git fetch** и **git merge**. Извлекает и загружает содержимое из удаленного репозитория и производит обновление локального репозитория этим содержимым)*
	 
17.  Просмотр списка настроенных удалённых репозиториев
     
     `git remote`
	 
	 *(показывает список удаленных подключений к другим репозиториям. При добавлении к команде **git remote** флага **-v** , выводит просмотр списка подключенных удаленных репозиториеев с указанием URL-адресов каждого подключения **git remote -v**)*
	 
18. Добавление удаленного репозитория к существующему локальному

     `git remote add`
	 
	 *(команда добавляет удаленный (внешний) репозиторий к существующему локальному **git remote add название удаленного репозитория ссылка на удаленный репозиторий**)*
	 
19. Отключение удаленного репозитория от локального

      `git remote remove`
	  
	  *(**git remote remove название удаленного репозитория** или сокращенно **git remote rm название удаленного репозитория** )*
	  
20. Изменение имени удаленного репозитория

      `git remote rename`
	  
	  *(меняет имя переданного удаленного репозитория **git remote rename старое имя удаленного репозитория новое имя удаленного репозитория**)*
	  
21. Просмотр всех удаленных репозиториев

      `git remote show`
	  
	  *(Выводит список всех подключенных удаленных репозиториев. Если после команды **git remote show** ввести имя репозитория, то выводит информацию об этом репозитории **git remote show имя удаленного репозитория**)*
