# инструкция по работе с git

## Список команд git

1. **git --version** - выводит версию установленной на компьютере программы git *(примечание: после git пробел и два знака тирэ(--)).*

2. **git init** - производит инициализацию рабочей папки в программе git (создает репозиторий) *(примечание: производится один раз для выбранной рабочей папки).*

3. **git status** - выводит статус документов находящихся в рабочей папке.

4. **git add название файла.расширение файла** - добавление файла к отслеживанию программой git *(примечание: название файла указывается полностью, с усказанием расширения файла. перед выполнением команды необходимо сохранить изменения в файле).*

5. **git add .** - добавление к отслеживанию программой git всех вайлов имеющихся в рабочей папке *(примечание: между add и . пробел. перед выполнением команды необходимо сохранить изменения в файлах)*

6. **git commit -m "коментарий"** - создание  т.н. "коммита" - новой версии документа с сохранением введенных изменений *(примечание: коментарий указывается в кавычках("))*.

7. **git commit -am "коментарий"** - команда объединяющая команды git add название файла.расширение файла и git commit -m "коментарий" *(примечание: коментарий указывается в кавычках("))*.

8. **git log** - выводит журнал сохраненых коммитов *(примечание: выводится полное наименование коммита, автор коммита, дата создания коммита, коментарий. слово HEAD обозначаеет в каком коммите мы находимся в данный момент, слово master обозначает какой коммит является актуальной версией документа)*.

9. **git reflog** - выводит список совершонных действий (сохранение коммитов, переходов между коммитами) *(примечание: выводится сокращенное наименование коммита  (первые семь символов), порядковый номер, коментарий к коммиту, пояснения из какого в какой коммит был совершен переход. слово HEAD обозначаеет в каком коммите мы находимся в данный момент, слово master обозначает какой коммит является актуальной версией документа)*

10. **git checkout наименование коммита** - открывает версию документа сохраненого в данном наименовании коммита *(примечание: в наименнование коммита достаточно ввести первые четыре символа).*

11. **git checkout master** - возвращает текущую, актуальную версию документа.

12. **git diff** - отображает разницу между текущей версией файла (с изменениями сохраненными в файле, но не сохранеными в коммит) и последней версией файла сохраненной в коммит *(примечание: красным цветом со знаком минус(-) отображаются строки которые были удалены, зеленым цветом со знаком плюс(+) строки которые были добавлены).*

13. **git diff наименование одного комита наименование другого комита** - отображает разницу между двумя выбранными комитами *(примечание: красным цветом со знаком минус(-) отображаются строки которые были удалены, зеленым цветом со знаком плюс(+) строки которые были добавлены. в наименнование коммита достаточно ввести первые четыре символа. наименование комитов пишутся друг за другом через пробел).*

14. **git branch** - выводит список веток

15. **git branch название_ветки** - создание новой ветки

16. **git checkout -b название_веки** - создание новой ветки и переход в нее 

17. **git merge название_ветки** - слияние веток

18. **git remote add origin ссылка_на _удаленный_репозиторий** - связь локального и удаленного репозитория 

19. **git push** - отправляет содержимое локального репозитьрия в удаленный репозитория

20. **git pull** - выгружает и делает слияние удаленного репозитория с локальным
